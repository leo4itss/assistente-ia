# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
npm run dev          # Vite dev server
npm run build        # Production build → dist/
npm run design:lint  # Validate DESIGN.md (must pass before committing design changes)
```

There is no `lint`, `test`, or `preview` script — do not invoke them.

Always run `npm run build` before committing to confirm no transpilation errors.

## Architecture

This is a **Figma Make**-generated React prototype for a configurable AI assistant interface. It has no router — screen switching is handled entirely by conditional rendering in `src/app/App.tsx`, which is both the root component and the source of truth for the `Assistant` interface (exported from there).

**Screen flow:**
- `showCreateAssistant` → renders `<CreateAssistantScreen>`
- `showCustomization` → renders `<CustomizationScreen assistant={selectedAssistant}>`
- Default → renders the chat shell (Sidebar + Header + ChatBar)

**State management:** `useState` + `localStorage` only. Cross-component sync uses a custom DOM event: `window.dispatchEvent(new Event("assistants-updated"))`. No global state library.

**Import alias:** `@/` maps to `src/` (configured in `vite.config.ts`).

**Figma asset resolver:** `import img from "figma:asset/<hash>.png"` is a custom Vite plugin that resolves to `src/assets/<hash>.png`. Never rename files in `src/assets/`.

## Design system

**`DESIGN.md` is the source of truth** for all design tokens (colors, typography, radius, spacing, components).

`src/styles/theme-design.css` applies those tokens as CSS variable overrides on top of shadcn's `theme.css`. The two files must stay in sync — edit `DESIGN.md` first, then update `theme-design.css` manually (the `@google/design.md` CLI does not yet support Tailwind v4 CSS export).

The project is **dark-first**: `background: #030712`, `surface: #111827`, `primary: #2563eb`. Components from `src/imports/` carry hardcoded hex values from Figma — check `DESIGN.md` before touching them.

Before creating a new UI component, check `src/app/components/ui/` — button, input, textarea, dialog, card, tabs, slider, and others already exist as shadcn/ui primitives.

## Files that must not be edited directly

| File / Folder | Reason |
|---|---|
| `src/imports/` | Auto-generated by Figma Make — SVG paths and design snapshots |
| `src/assets/` | Assets identified by content hash — renaming breaks `figma:asset/` imports |
| `src/styles/theme.css` | Mirror of `default_shadcn_theme.css`; editing breaks the entire design system |
| `src/styles/theme-design.css` | Driven by `DESIGN.md` — edit the source doc, not this file directly |
| `default_shadcn_theme.css` | Kept in sync with Figma Make (`KEEP_IN_SYNC`) |
| `vite.config.ts` | Contains the `figma:asset/` resolver — removing it breaks all asset imports |
| `vercel.json` | Vercel deploy config (forces npm, sets output dir) |

**MUI 7 is in `dependencies` but is not used.** Do not add new MUI components.

## Conventions

**Naming:** component files `PascalCase.tsx`; shadcn/ui in `ui/` use `kebab-case.tsx`; no `index.ts` barrel files — import by direct path.

**Types:** The `Assistant` interface lives in `src/app/App.tsx` (exported). Local types stay in their own component file. No `types/` folder.

**Commits:** use `feat/fix/chore: description [model]` format (e.g. `feat: adiciona contador [claude]`). Branch from `dev`; push to `dev` triggers auto-deploy on Vercel. PRs `dev → main` only for stable releases.

See `AGENTS.md` for the full conventions reference and `DESIGN.md` for the complete token specification.
